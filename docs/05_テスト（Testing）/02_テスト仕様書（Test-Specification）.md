## 1. 概要

本文書は、Construction-GRC-Systemの各機能モジュールに対するテスト仕様を定める。リスク管理（RSK）、コンプライアンス管理（CMP）、統制管理（CTL）、監査管理（AUD）、レポート管理（RPT）の各モジュールについて、テスト観点・テスト条件・期待結果を明確化する。

---

## 2. 共通テスト仕様

### 2.1 認証・認可テスト

| テスト項目 | テスト条件 | 期待結果 |
|-----------|----------|---------|
| ログイン（正常） | 有効なメールアドレスとパスワードを入力 | JWTトークンが発行され、ダッシュボードに遷移する |
| ログイン（無効な認証情報） | 誤ったパスワードを入力 | 401エラーが返され、エラーメッセージが表示される |
| ログイン（アカウントロック） | 5回連続でログイン失敗 | アカウントが一時ロックされ、ロックメッセージが表示される |
| トークン有効期限 | アクセストークンの有効期限が切れた状態でAPIを呼び出す | リフレッシュトークンで自動更新される |
| リフレッシュトークン期限切れ | リフレッシュトークンの有効期限が切れた状態 | ログイン画面にリダイレクトされる |
| 権限不足アクセス | 権限のないリソースにアクセス | 403エラーが返される |
| 未認証アクセス | 認証なしでAPIを呼び出す | 401エラーが返される |
| パスワード変更 | 現在のパスワードと新パスワードを入力 | パスワードが更新され、既存セッションが無効化される |
| パスワードリセット | メールアドレスを入力してリセット要求 | リセットメールが送信される |

### 2.2 RBAC テスト

| ロール | 許可操作 | テスト内容 |
|--------|---------|----------|
| システム管理者 | 全機能の操作 | 全CRUD操作が実行可能であること |
| リスク管理者 | RSKモジュールの全操作、他モジュールの参照 | RSKの作成・編集・削除が可能、CMPは参照のみ |
| コンプライアンス管理者 | CMPモジュールの全操作、他モジュールの参照 | CMPの作成・編集・削除が可能、RSKは参照のみ |
| 監査担当者 | AUDモジュールの全操作、RSK/CMP/CTLの参照 | AUDの作成・編集が可能、他は参照のみ |
| 一般ユーザー | 全モジュールの参照、一部データ入力 | 参照操作のみ可能、削除は不可 |
| 閲覧者 | 全モジュールの参照のみ | 参照のみ可能、作成・編集・削除は全て不可 |

### 2.3 共通UIテスト

| テスト項目 | テスト条件 | 期待結果 |
|-----------|----------|---------|
| ページネーション | 一覧画面で大量データを表示 | 指定件数ごとにページ分割され、ページ切替が動作する |
| ソート | 一覧画面のカラムヘッダーをクリック | 昇順/降順でソートされる |
| フィルター | フィルター条件を設定 | 条件に合致するデータのみ表示される |
| 検索 | キーワードを入力して検索 | キーワードを含むデータが表示される |
| バリデーション（必須項目） | 必須項目を空のまま送信 | エラーメッセージが表示され、送信されない |
| バリデーション（文字数制限） | 最大文字数を超える入力 | エラーメッセージが表示される |
| 確認ダイアログ | 削除ボタンをクリック | 確認ダイアログが表示され、キャンセル可能 |
| ローディング表示 | データ取得中 | ローディングインジケーターが表示される |
| エラー表示 | API呼び出しが失敗 | エラースナックバーが表示される |
| 空状態表示 | データが0件の場合 | 「データがありません」等の空状態メッセージが表示される |

## 3. リスク管理（RSK）テスト仕様

### 3.1 リスク登録

| テスト項目 | テスト条件 | 期待結果 |
|-----------|----------|---------|
| 正常登録 | 全必須項目を入力して保存 | リスクが登録され、一覧に表示される |
| 必須項目未入力 | タイトルを空にして保存 | バリデーションエラーが表示される |
| リスクレベル自動計算 | 発生確率と影響度を入力 | リスクスコアが自動計算され、リスクレベルが設定される |
| カテゴリ設定 | リスクカテゴリを選択 | 選択したカテゴリでリスクが登録される |
| 添付ファイル | 関連資料をアップロード | ファイルが添付され、ダウンロード可能 |
| 添付ファイル制限 | 10MBを超えるファイルをアップロード | エラーメッセージが表示され、アップロードされない |
| 担当者アサイン | リスク対応担当者を選択 | 担当者にアサインされ、通知が送信される |
| 重複チェック | 同一タイトルのリスクを登録 | 警告メッセージが表示される（登録は可能） |

### 3.2 リスク評価

| テスト項目 | テスト条件 | 期待結果 |
|-----------|----------|---------|
| 定量評価 | 発生確率（0.0-1.0）と影響度（1-10）を入力 | リスクスコアが計算される（確率 x 影響度） |
| 発生確率の境界値 | 0.0, 0.5, 1.0 を入力 | 各値で正しく計算される |
| 発生確率の範囲外 | -0.1, 1.1 を入力 | バリデーションエラーが表示される |
| 影響度の境界値 | 1, 5, 10 を入力 | 各値で正しく計算される |
| リスクマトリクス表示 | 複数リスクの評価データが存在 | ヒートマップ形式でリスク分布が表示される |
| 評価履歴 | リスクの評価値を変更 | 変更前の評価値が履歴として保存される |
| 一括再評価 | 複数リスクを選択して一括再評価 | 選択したリスクの評価が更新される |

### 3.3 リスクモニタリング

| テスト項目 | テスト条件 | 期待結果 |
|-----------|----------|---------|
| ステータス更新 | リスクのステータスを変更 | ステータスが更新され、変更履歴が記録される |
| 期限アラート | 対応期限の7日前 | アラート通知が送信される |
| 期限超過 | 対応期限を過ぎたリスク | ダッシュボードに警告表示される |
| エスカレーション | リスクレベルがCRITICALに変更 | 上位管理者に自動通知される |
| トレンド表示 | 過去のリスクデータが存在 | 時系列でリスクの推移がグラフ表示される |

### 3.4 リスク対応

| テスト項目 | テスト条件 | 期待結果 |
|-----------|----------|---------|
| 対応策登録 | リスクに対する対応策を登録 | 対応策がリスクに紐付けて保存される |
| 対応策の進捗管理 | 対応策の進捗率を更新 | 進捗率が更新され、ダッシュボードに反映される |
| 対応策の完了 | 全ての対応策を完了に変更 | リスクのステータスを「対応済」に変更可能になる |
| コメント追加 | リスクにコメントを追加 | コメントが保存され、関係者に通知される |

## 4. コンプライアンス管理（CMP）テスト仕様

### 4.1 法令・規制管理

| テスト項目 | テスト条件 | 期待結果 |
|-----------|----------|---------|
| 法令登録 | 法令名、条文番号、施行日を入力 | 法令が登録される |
| 法令検索 | キーワードで法令を検索 | 全文検索結果が表示される |
| 法令更新通知 | 法令の内容を更新 | 関連するコンプライアンス項目の担当者に通知される |
| 法令のバージョン管理 | 法令を更新 | 更新前の版が履歴として保存される |
| 法令と規制のマッピング | 法令に関連する規制を紐付け | 関連性が保存され、参照可能 |

### 4.2 遵守状況管理

| テスト項目 | テスト条件 | 期待結果 |
|-----------|----------|---------|
| 遵守項目登録 | 遵守すべき項目と基準を入力 | 遵守項目が登録される |
| 遵守状況記録 | 遵守/非遵守/確認中のステータスを設定 | ステータスが記録される |
| 非遵守時の是正措置 | 非遵守の場合に是正措置を登録 | 是正措置が登録され、期限が設定される |
| 遵守率の集計 | 複数の遵守項目が存在 | 遵守率が自動計算される |
| 部門別遵守状況 | 複数部門のデータが存在 | 部門別の遵守率が表示される |
| 遵守チェックリスト | 定期チェック対象の項目一覧 | チェックリスト形式で確認・記録可能 |

### 4.3 期限管理

| テスト項目 | テスト条件 | 期待結果 |
|-----------|----------|---------|
| 期限設定 | 遵守項目に期限を設定 | 期限が保存される |
| 期限通知（30日前） | 期限の30日前 | 事前通知が送信される |
| 期限通知（7日前） | 期限の7日前 | 緊急通知が送信される |
| 期限通知（当日） | 期限当日 | 期限到来通知が送信される |
| 期限超過 | 期限を超過 | 超過アラートが表示され、エスカレーションされる |
| 期限の延長 | 期限を変更 | 変更理由の入力が必須、履歴が記録される |

### 4.4 教育・研修管理

| テスト項目 | テスト条件 | 期待結果 |
|-----------|----------|---------|
| 研修登録 | コンプライアンス研修を登録 | 研修情報が登録される |
| 受講記録 | 受講完了を記録 | 受講者の完了状態が更新される |
| 未受講者一覧 | 研修対象者に未受講者がいる場合 | 未受講者一覧が表示される |
| 受講催促通知 | 研修期限の7日前に未受講 | 催促通知が送信される |
