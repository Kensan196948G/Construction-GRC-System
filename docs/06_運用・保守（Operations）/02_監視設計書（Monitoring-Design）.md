## 1. 概要

### 1.1 目的

本文書は、Construction-GRC-Systemの監視設計について、アプリケーション監視、インフラ監視、ビジネスメトリクス監視、およびアラート設計を定義する。Prometheus / Grafanaを中心とした統合監視基盤により、システムの可用性・性能・セキュリティを継続的に監視する。

### 2.1 アプリケーションメトリクス

| メトリクス名 | 説明 | 収集方法 | 収集間隔 |
|-------------|------|---------|---------|
| `http_requests_total` | HTTPリクエスト総数 | Django Middleware | 15秒 |
| `http_request_duration_seconds` | リクエスト処理時間 | Django Middleware | 15秒 |

**閾値設定:**

| 条件 | Severity | 説明 |
|------|----------|------|
| 平均レスポンスタイム > 1秒（5分間） | Warning | パフォーマンス劣化の兆候 |
| 平均レスポンスタイム > 3秒（5分間） | Critical | ユーザ体験に重大な影響 |
| 5xxエラー率 > 1%（5分間） | Warning | サーバーエラーの発生 |
| 5xxエラー率 > 5%（5分間） | Critical | 深刻なサーバー障害 |
| リクエスト数が通常の200%超 | Warning | 異常アクセスの可能性 |

## 5. アラート設計

### 5.1 Severity分類

| Severity | レベル | 説明 | 対応時間 | 通知方法 |
|----------|-------|------|---------|---------|
| P1-Critical | 緊急 | サービス全面停止、データ損失リスク | 即時対応 | Slack + 電話 + メール |
| P2-High | 重大 | 主要機能の停止、性能著しい劣化 | 30分以内 | Slack + 電話 + メール |
| P3-Medium | 中度 | 一部機能の障害、性能低下 | 4時間以内 | Slack + メール |
| P4-Low | 軽微 | 軽微な問題、情報通知 | 次営業日 | Slack |
| P5-Info | 情報 | 情報提供、トレンド通知 | 対応不要 | Slack |

### 5.2 アラートルール定義

#### 5.2.1 P1-Critical アラート

| アラート名 | 条件 | 持続時間 |
|-----------|------|---------|
| ServiceDown | 全ヘルスチェックエンドポイント応答なし | 1分 |
| DatabaseDown | PostgreSQL Primary接続不可 | 30秒 |
| DataCorruption | データ整合性チェック失敗 | 即時 |
| SecurityBreach | 不正アクセス検知（大量認証失敗） | 即時 |
| DiskFull | ディスク使用率 > 95% | 即時 |

#### 5.2.2 P2-High アラート

| アラート名 | 条件 | 持続時間 |
|-----------|------|---------|
| HighErrorRate | 5xxエラー率 > 5%（5分間） | 5分 |
| DatabaseReplicationLag | レプリケーション遅延 > 60秒 | 5分 |
| HighResponseTime | 平均レスポンスタイム > 3秒 | 5分 |
| MemoryPressure | メモリ使用率 > 90% | 5分 |
| CeleryQueueBacklog | タスクキュー滞留 > 1000件 | 10分 |
| SSLCertExpiringSoon | SSL証明書有効期限 < 7日 | - |

#### 5.2.3 P3-Medium アラート

| アラート名 | 条件 | 持続時間 |
|-----------|------|---------|
| ElevatedErrorRate | 5xxエラー率 > 1%（5分間） | 10分 |
| SlowQueries | スロークエリ > 10件/分 | 10分 |
| HighCPU | CPU使用率 > 70%（5分平均） | 15分 |
| DiskSpaceWarning | ディスク使用率 > 80% | - |
| BackupFailure | バックアップジョブ失敗 | 即時 |
| BatchJobFailure | 定期バッチ失敗 | 即時 |

#### 5.2.4 P4-Low アラート

| アラート名 | 条件 | 持続時間 |
|-----------|------|---------|
| HighConnectionCount | DB接続数 > 70% | 30分 |
| CacheHitRateLow | キャッシュヒット率 < 90% | 30分 |
| LogVolumeHigh | ログ出力量が通常の200% | 30分 |
| SSLCertExpiring30d | SSL証明書有効期限 < 30日 | - |

### 5.3 アラート通知設定

#### 5.3.1 Slack通知

```yaml
# AlertManager設定例
receivers:
  - name: 'critical-alerts'
    slack_configs:
      - channel: '#grc-alerts-critical'
        send_resolved: true
        title: '[{{ .Status | toUpper }}] {{ .CommonLabels.alertname }}'
        text: >-
          *Summary:* {{ .CommonAnnotations.summary }}
          *Description:* {{ .CommonAnnotations.description }}
          *Severity:* {{ .CommonLabels.severity }}
          *Instance:* {{ .CommonLabels.instance }}

  - name: 'warning-alerts'
    slack_configs:
      - channel: '#grc-alerts-warning'
        send_resolved: true

  - name: 'info-alerts'
    slack_configs:
      - channel: '#grc-alerts-info'
        send_resolved: false
```

#### 5.3.2 メール通知

| 条件 | 宛先 |
|------|------|
| P1-Critical | 運用チーム全員 + 管理者 + PM |
| P2-High | 運用チーム全員 + 管理者 |
| P3-Medium | 運用当番 |
| P4-Low | 運用メーリングリスト（日次ダイジェスト） |

### 5.4 アラート抑制ルール

| ルール | 条件 | 目的 |
|--------|------|------|
| メンテナンスウィンドウ | 計画停止期間中 | 計画メンテナンス中の誤報防止 |
| インヒビション | 上位障害発生時の下位アラート | アラートストーム防止 |
| 重複排除 | 同一アラートの連続発報 | ノイズ軽減（5分間隔） |
| 営業時間外 | P4-LowのSlack通知 | 夜間の不要通知抑制 |

## 7. ログ管理

### 7.1 ログ収集アーキテクチャ

```
アプリケーション / インフラ
    │
    ▼
Promtail（ログ収集エージェント）
    │
    ▼
Loki（ログストレージ）
    │
    ▼
Grafana（ログ検索・可視化）
```

### 7.2 ログレベル定義

| レベル | 用途 | 保持期間 |
|--------|------|---------|
| CRITICAL | システム停止レベル | 1年 |
| ERROR | エラー（復旧可能） | 6ヶ月 |
| WARNING | 警告・注意 | 3ヶ月 |
| INFO | 情報・正常動作 | 1ヶ月 |
| DEBUG | デバッグ情報 | 7日（本番では原則無効） |
