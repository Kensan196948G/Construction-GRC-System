## 1. アーキテクチャ概要

### 1.1 アーキテクチャスタイル

本システムは**モノリシック・レイヤードアーキテクチャ**を採用する。将来的なマイクロサービス分離を見据え、Djangoアプリケーション単位で疎結合な設計とする。

### 1.2 アーキテクチャ原則

| 原則 | 説明 |
|------|------|
| **関心の分離** | プレゼンテーション/ビジネスロジック/データアクセスを明確に分離 |
| **疎結合** | Djangoアプリ間はサービス層を介して連携し、直接的なモデル参照を最小化 |
| **API First** | フロントエンド-バックエンド間はREST APIで通信 |
| **セキュリティバイデザイン** | 設計段階からセキュリティを組み込み |
| **12 Factor App** | 環境変数による設定管理、ステートレスなプロセス設計 |

    ├── test_models.py

**サービス層の役割:**

```python
# apps/risks/services.py（設計例）
class RiskService:
    @staticmethod
    def create_risk(data: dict, user: User) -> Risk:
        """リスクを作成し、関連処理を実行する"""
        # 1. バリデーション
        # 2. リスクID自動採番
        # 3. リスクスコア計算
        # 4. データベース保存
        # 5. 通知送信（Celeryタスク）
        # 6. 監査ログ記録
        pass

    @staticmethod
    def evaluate_risk(risk_id: int, evaluation_data: dict, user: User) -> RiskEvaluation:
        """リスク評価を実施する"""
        # 1. 固有リスクスコア計算
        # 2. 残留リスクスコア計算
        # 3. リスクレベル判定
        # 4. ヒートマップ更新
        # 5. 高リスクアラート送信
        pass
```

**カスタムマネージャの活用:**

```python
# apps/risks/managers.py（設計例）
class RiskManager(models.Manager):
    def active(self):
        """論理削除されていないリスクを返す"""
        return self.filter(is_deleted=False)

    def high_risk(self):
        """高リスク以上のリスクを返す"""
        return self.active().filter(risk_level__in=['critical', 'high'])
```

## 7. 可用性・冗長性設計

### 7.1 SLA目標

| 指標 | 目標値 |
|------|-------|
| 可用性 | 99.9%（月間ダウンタイム最大43分） |
| RTO（目標復旧時間） | 1時間 |
| RPO（目標復旧地点） | 5分（WALレプリケーション） |

### 7.2 冗長性構成

| コンポーネント | 冗長性方式 |
|-------------|----------|
| Webサーバー（Nginx） | 2台以上のActive-Active |
| アプリケーション（Gunicorn） | 3台以上のステートレスインスタンス |
| データベース（PostgreSQL） | Primary + Standby（ストリーミングレプリケーション） |
| キャッシュ（Redis） | Redis Sentinel による自動フェイルオーバー |
| ワーカー（Celery） | 複数ワーカーによる分散処理 |
| ファイルストレージ | Azure Blob Storage（GRS冗長） |

### 7.3 バックアップ戦略

| 種別 | 頻度 | 保持期間 | 方法 |
|------|------|---------|------|
| フルバックアップ | 日次 | 30日 | pg_dump + Azure Blob Storage |
| 差分バックアップ | 6時間ごと | 7日 | WALアーカイブ |
| ポイントインタイムリカバリ | 連続 | 7日 | PostgreSQL PITR |
| ファイルバックアップ | 日次 | 90日 | Azure Blob Storage（GRS） |
