## 1. 情報セキュリティ基本方針

### 1.1 基本理念

Construction-GRC-Systemは、建設業における統合リスク＆コンプライアンス管理を担うシステムとして、取り扱う情報の機密性、完全性、可用性を確保するために、ISO/IEC 27001:2022およびNIST CSF 2.0に準拠した情報セキュリティマネジメントを実施する。

### 1.2 基本方針

1. **情報資産の保護:** システムで管理する全ての情報資産（リスク評価データ、コンプライアンス記録、監査証跡、個人情報、建設現場安全データ）を適切に保護する
2. **法令遵守:** 個人情報保護法、不正アクセス禁止法、建設業法、品確法、労安法その他関連法令を遵守する
3. **リスクベースアプローチ:** リスクアセスメントに基づき、適切なセキュリティ対策を選択・実施する
4. **継続的改善:** PDCAサイクルにより、情報セキュリティマネジメントを継続的に改善する
5. **全員参加:** システム利用者およびプロジェクト関係者全員が情報セキュリティに対する責任を認識し、セキュリティ意識の向上に努める

## 3. セキュリティ目標

### 3.1 定量目標

| 目標 | 指標 | 目標値 | 測定頻度 |
|------|------|--------|---------|
| セキュリティインシデント | 重大インシデント発生件数 | 0件/年 | 月次 |
| 脆弱性対応 | Critical脆弱性の24時間以内対応率 | 100% | 月次 |
| 脆弱性対応 | High脆弱性の72時間以内対応率 | 100% | 月次 |
| アクセス制御 | 不正アクセス検知率 | 99%以上 | 四半期 |
| 可用性 | システム稼働率 | 99.5%以上 | 月次 |
| 教育 | セキュリティ教育受講率 | 100% | 年次 |
| 監査 | 監査指摘事項の期限内是正率 | 95%以上 | 四半期 |

### 3.2 定性目標

- ISO/IEC 27001:2022認証の取得・維持に必要なセキュリティ水準の確保
- NIST CSF 2.0の全6機能（統治、識別、防御、検知、対応、復旧）の実装
- 建設業特有の法令要件（建設業法、品確法、労安法）に対応するデータ保護の実現
- セキュリティインシデントの迅速な検知・対応体制の確立

## 5. セキュリティ管理策

### 5.1 管理策の概要

本システムでは、ISO/IEC 27001:2022 附属書Aの93管理策を基盤とし、以下のカテゴリで管理策を実装する。

| カテゴリ | 管理策数 | 関連文書 |
|---------|---------|---------|
| 組織的管理策（A.5） | 37 | 本文書、GRC-GS-001 |
| 人的管理策（A.6） | 8 | 本文書 |
| 物理的管理策（A.7） | 14 | インフラ設計書 |
| 技術的管理策（A.8） | 34 | GRC-SEC-002, GRC-SEC-003 |

### 5.2 主要セキュリティ管理策

| 領域 | 管理策 | 実装方法 |
|------|--------|---------|
| アクセス制御 | RBAC + JWT + MFA | GRC-SEC-002参照 |
| 暗号化 | 通信: TLS 1.3、保存: AES-256 | 全通信のHTTPS化、DB暗号化 |
| ログ・監視 | 全操作の監査ログ | Prometheus + Grafana + Loki |
| バックアップ | 日次3世代管理 | GRC-OPS-004参照 |
| 脆弱性管理 | 定期スキャン + パッチ管理 | GRC-SEC-003参照 |
| インシデント対応 | CSIRT体制 + 対応フロー | GRC-SEC-004参照 |
| セキュアな開発 | SAST/DAST + コードレビュー | CI/CDパイプライン統合 |
| ネットワーク | ファイアウォール + セグメンテーション | DMZ構成、セグメント分離 |

## 6. 情報分類と取り扱い

### 6.1 情報分類基準

| 分類レベル | 定義 | 取り扱い基準 |
|-----------|------|-------------|
| 極秘 | 漏洩時に重大な損害が生じる情報 | アクセス制限（need-to-know）、暗号化必須、印刷制限 |
| 機密 | 漏洩時に相当な損害が生じる情報 | アクセス制限、暗号化推奨 |
| 社外秘 | 社外への開示が制限される情報 | 社外持ち出し制限 |
| 公開 | 制限なく公開可能な情報 | 制限なし |

### 6.2 データライフサイクル管理

| フェーズ | セキュリティ要件 |
|---------|----------------|
| 作成 | 分類ラベルの付与、アクセス権の設定 |
| 保存 | 暗号化、アクセス制御、バックアップ |
| 利用 | 認証・認可に基づくアクセス、監査ログ |
| 共有 | 暗号化通信、アクセス権の確認 |
| アーカイブ | 長期保存、暗号化、アクセス制限 |
| 廃棄 | 安全な削除（上書き消去/暗号鍵破棄） |
