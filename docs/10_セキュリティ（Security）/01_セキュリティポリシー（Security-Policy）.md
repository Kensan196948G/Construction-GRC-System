## 1. 情報セキュリティ基本方針

### 1.1 方針声明

当社は、建設業統合リスク＆コンプライアンス管理システム（Construction-GRC-System）において取り扱う全ての情報資産を、不正アクセス、漏洩、改ざん、破壊等の脅威から保護し、情報の機密性（Confidentiality）、完全性（Integrity）、可用性（Availability）を確保する。

ISO/IEC 27001:2022に定める全93管理策およびNIST Cybersecurity Framework 2.0の6機能（統治・特定・防御・検知・対応・復旧）に準拠した情報セキュリティマネジメントシステム（ISMS）を構築・運用し、建設業法・品確法・労安法等の関連法令を遵守しつつ、継続的な改善を推進する。

### 1.2 基本方針の目的

| # | 目的 | 説明 | 関連規格 |
|---|------|------|----------|
| 1 | 情報資産の保護 | GRCシステムで管理する全ての情報資産を適切に保護する | ISO27001 A.5.1 |
| 2 | 法令・規格の遵守 | ISO27001全93管理策、NIST CSF 2.0、建設業関連法令等の要件を遵守する | ISO27001 A.5.31 / NIST GV.OC |
| 3 | 事業継続性の確保 | セキュリティインシデント発生時においても事業継続性を確保する | ISO27001 A.5.29, A.5.30 / NIST RC |
| 4 | 利害関係者の信頼確保 | 顧客、パートナー、規制当局等の利害関係者からの信頼を維持する | NIST GV.SC |
| 5 | セキュリティ文化の醸成 | 組織全体でのセキュリティ意識の向上と定着を図る | ISO27001 A.6.3 / NIST GV.AT |

### 1.3 セキュリティ方針の原則

| 原則 | 説明 | 適用例 |
|------|------|--------|
| 機密性（Confidentiality） | 情報は許可された者のみがアクセスできる状態を維持する | アクセス制御、暗号化 |
| 完全性（Integrity） | 情報及び処理方法の正確さ及び完全さを保護する | ハッシュ検証、改ざん検知 |
| 可用性（Availability） | 許可された利用者が必要なときに情報にアクセスできることを確保する | 冗長構成、バックアップ |
| 最小権限の原則 | ユーザーには業務に必要な最小限の権限のみを付与する | RBAC設計 |
| 職務分離 | 重要な業務は複数人で分担し、不正リスクを低減する | 承認フロー |
| 多層防御（Defense in Depth） | 単一の対策に依存せず、複数層の防御を実装する | WAF + アプリ + DB |
| ゼロトラスト | 全てのアクセスを検証し、暗黙の信頼を排除する | 都度認証・認可 |

## 7. 建設業法関連セキュリティ要件

| 品確法 | 技術提案情報の保護 | 情報分類、アクセス制御、暗号化 | A.5.12, A.5.15 |
| 労安法 | 健康診断データ等の個人情報保護 | PII保護、データマスキング | A.5.34, A.8.11 |
| 個人情報保護法 | 個人情報の適正な取扱い | 同意管理、影響評価、データ主体権利対応 | A.5.34 |

### 7.2 建設業固有のセキュリティリスク

| リスク | 影響 | 対策 |
|--------|------|------|
| 入札情報の漏洩 | 公正な競争の阻害、行政処分 | 極秘情報として暗号化管理、アクセスログ監視 |
| 経審データの改ざん | 不正な評点取得、許可取消 | 完全性チェック、変更履歴管理 |
| 技術者資格情報の不正利用 | 無資格配置、業法違反 | アクセス制御、操作ログ記録 |
| 安全衛生データの漏洩 | プライバシー侵害、訴訟リスク | PII保護、マスキング、暗号化 |

## 9. 違反への対応

### 9.1 違反レベルと対応

| 違反レベル | 例 | 対応措置 | エスカレーション先 |
