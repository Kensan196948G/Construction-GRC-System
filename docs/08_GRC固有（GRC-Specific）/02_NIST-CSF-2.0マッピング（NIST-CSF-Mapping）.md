## 1. 目的

本文書は、NIST Cybersecurity Framework（CSF）2.0の6つの機能（GOVERN、IDENTIFY、PROTECT、DETECT、RESPOND、RECOVER）と、ISO/IEC 27001:2022管理策とのクロスマッピングを定義する。建設業GRCシステムにおける包括的なサイバーセキュリティ管理の実現を支援することを目的とする。

---

## 2. NIST CSF 2.0の概要

### 2.1 フレームワーク構成

NIST CSF 2.0は以下の6つの機能（Function）で構成される。

```
                    ┌──────────────┐
                    │   GOVERN     │
                    │  （統治）     │
                    └──────┬───────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
   ┌────▼────┐       ┌────▼────┐       ┌────▼────┐
   │IDENTIFY │       │ PROTECT │       │ DETECT  │
   │（識別） │       │（保護） │       │（検知） │
   └────┬────┘       └────┬────┘       └────┬────┘
        │                  │                  │
        │            ┌─────▼─────┐            │
        │            │  RESPOND  │            │
        │            │ （対応）  │            │
        │            └─────┬─────┘            │
        │                  │                  │
        │            ┌─────▼─────┐            │
        └────────────┤  RECOVER  ├────────────┘
                     │ （復旧）  │
                     └───────────┘
```

### 2.2 機能一覧

| 機能 | コード | 説明 | カテゴリ数 |
|------|--------|------|----------|
| 統治（Govern） | GV | 組織のサイバーセキュリティリスク管理の戦略、期待、方針の確立と監視 | 6 |
| 識別（Identify） | ID | 組織のサイバーセキュリティリスクの理解 | 4 |
| 保護（Protect） | PR | サイバーセキュリティリスクを管理するためのセーフガードの使用 | 5 |
| 検知（Detect） | DE | サイバーセキュリティ攻撃やイベントの発見と分析 | 2 |
| 対応（Respond） | RS | 検知されたサイバーセキュリティインシデントへの対処 | 4 |
| 復旧（Recover） | RC | サイバーセキュリティインシデントの影響を受けた資産・運用の復元 | 2 |
