## 2. プロジェクトの目的

### 2.1 背景

みらい建設工業は、年間売上高約800億円、従業員数約2,500名を擁する総合建設会社である。近年、以下の経営課題が顕在化している。

| # | 経営課題 | 影響度 | 緊急度 |
|---|----------|--------|--------|
| 1 | 建設業法改正への対応遅延リスク | 高 | 高 |
| 2 | ISO27001認証維持に必要な管理策の属人化 | 高 | 中 |
| 3 | 労働安全衛生法に基づく安全管理の分散管理 | 高 | 高 |
| 4 | 品確法対応のための品質記録管理の非効率 | 中 | 中 |
| 5 | 内部監査業務の工数増大（年間約1,200人時） | 中 | 低 |
| 6 | 各種法令改正への追従体制の脆弱性 | 高 | 中 |
| 7 | グループ会社間でのリスク情報共有の欠如 | 中 | 中 |

### 2.2 目的

本プロジェクトは、上記課題を解決するため、以下を実現する統合GRC（Governance, Risk, Compliance）システムを構築する。

1. **ガバナンス強化** - 組織横断的なリスク可視化と意思決定支援の実現
2. **リスク管理高度化** - ISO31000準拠のリスクアセスメントプロセスの自動化
3. **コンプライアンス徹底** - 建設業法・品確法・労安法等の法令遵守状況の一元管理
4. **情報セキュリティ** - ISO27001:2022全93管理策の実装状況追跡と継続的改善
5. **監査効率化** - 内部監査業務の50%以上の工数削減
6. **NIST CSF 2.0対応** - サイバーセキュリティフレームワークへの準拠

### 2.3 ビジョンステートメント

> 「建設業界におけるGRC管理のデファクトスタンダードとなるシステムを構築し、みらい建設工業グループ全体のガバナンス・リスク管理・コンプライアンスを統合的に強化する」

## 3. スコープ

### 3.1 スコープ内（In Scope）

| カテゴリ | 対象範囲 |
|----------|----------|
| **リスク管理** | リスク台帳管理、リスクアセスメント、リスク対応計画、リスクモニタリング、リスクレポーティング |
| **コンプライアンス管理** | 法令管理台帳、遵守状況評価、法改正追跡、是正措置管理 |
| **ISO27001管理** | 全93管理策の実装・運用・監視、適用宣言書(SoA)管理、管理策有効性評価 |
| **NIST CSF 2.0** | 6機能（統治・識別・防御・検知・対応・復旧）の成熟度評価・改善管理 |
| **建設業法対応** | 技術者配置管理、下請契約管理、施工体制台帳管理 |
| **品確法対応** | 品質記録管理、検査記録追跡、品質指標モニタリング |
| **労安法対応** | 安全衛生計画管理、事故報告・分析、安全パトロール記録 |
| **内部監査** | 監査計画策定、監査実施支援、指摘事項管理、フォローアップ |
| **レポーティング** | 経営ダッシュボード、定期報告書自動生成、規制当局向けレポート |
| **外部連携** | 既存基幹システム連携、外部データソース連携 |

### 3.2 スコープ外（Out of Scope）

| 項目 | 理由 |
|------|------|
| 基幹系システム（ERP）の刷新 | 別プロジェクトとして計画中 |
| BIM/CIMシステムとの直接連携 | Phase5以降で検討 |
| 海外拠点への展開 | 国内拠点安定稼働後に検討 |
| 会計・財務リスク管理 | 既存会計システムの機能で対応 |
| 物理セキュリティ管理システム | 別途設備管理部門で対応 |

## 4. 成功基準（Success Criteria）

### 4.1 定量的成功基準

| # | 成功基準 | 目標値 | 測定方法 |
|---|----------|--------|----------|
| 1 | ISO27001管理策の自動評価率 | 80%以上 | システム自動評価数 / 全93管理策 |
| 2 | 内部監査工数削減率 | 50%以上 | 導入前後の監査工数比較 |
| 3 | リスク特定から対応計画策定までのリードタイム | 5営業日以内 | システムログ分析 |
| 4 | コンプライアンス違反インシデント数 | 前年比30%削減 | インシデント管理台帳 |
| 5 | システム稼働率 | 99.5%以上 | 運用監視ツール |
| 6 | ユーザー満足度 | 4.0/5.0以上 | 利用者アンケート |
| 7 | 法改正対応リードタイム | 30日以内 | 法改正通知〜システム反映日数 |
| 8 | レポート生成自動化率 | 70%以上 | 自動生成レポート数 / 全レポート数 |

### 4.2 定性的成功基準

- 経営層がリアルタイムでリスク状況を把握できる環境が整備されること
- 各部門のリスク・コンプライアンス担当者が日常業務の中で自然にシステムを活用できること
- 外部監査において、GRCシステムの有効性について肯定的な評価を得ること
- 建設業法・品確法・労安法の遵守状況を一元的に可視化できること
