### 3.2 M2：要件定義完了

| 項目 | 内容 |
|------|------|
| **計画日** | 2026年6月15日 |
| **目的** | Phase1の機能・非機能要件を確定し、設計・開発の基準を確立する |

**ゲートレビュー基準：**

| # | レビュー項目 | 判定基準 | 合否 |
|---|------------|---------|------|
| 1 | 機能要件定義書の完成度 | 全機能の要件が定義され、ステークホルダーの承認を得ていること | |
| 2 | 非機能要件定義書の完成度 | 性能、セキュリティ、可用性等の基準が数値で定義されていること | |
| 3 | 画面設計の完成度 | 主要画面のワイヤーフレームが作成され、業務チームの合意を得ていること | |
| 4 | ステークホルダーの合意 | 要件レビュー会議で全主要ステークホルダーの承認を得ていること | |
| 5 | トレーサビリティ | 要件からWBSタスクへのトレーサビリティが確保されていること | |
| 6 | 見積精度 | 要件に基づく開発工数の再見積が実施されていること | |

**Go/No-Go判定：**

| 判定 | 条件 |
|------|------|
| **Go** | 上記6項目中、#1,#2,#4が全て合格、かつ残りの項目に重大な未解決事項がないこと |
| **Conditional Go** | #1,#2,#4に軽微な未解決事項があるが、2週間以内に解消見込みがあること |
| **No-Go** | #1,#2,#4のいずれかに重大な未解決事項がある場合。対策計画を策定し、再レビューを実施 |

**判定者：** PM、CIO、情報システム部長

### 3.3 M3：Phase1（基盤構築）完了

| 項目 | 内容 |
|------|------|
| **計画日** | 2026年9月30日 |
| **目的** | 共通基盤が完成し、Phase2以降の開発が開始可能な状態を確認する |

**ゲートレビュー基準：**

| # | レビュー項目 | 判定基準 | 重要度 |
|---|------------|---------|--------|
| 1 | 認証・認可機能 | SSO/MFA/RBACが正常動作し、セキュリティテスト合格 | 必須 |
| 2 | 共通マスタ管理 | 全マスタのCRUD操作が正常動作 | 必須 |
| 3 | API基盤 | API Gateway経由の全APIが正常動作 | 必須 |
| 4 | UI基盤 | デザインシステムに準拠した画面が表示 | 必須 |
| 5 | インフラ | 本番・ステージング環境が安定稼働 | 必須 |
| 6 | CI/CD | 自動ビルド・テスト・デプロイが正常動作 | 必須 |
| 7 | テストカバレッジ | 80%以上 | 必須 |
| 8 | セキュリティ | Critical/High脆弱性がゼロ | 必須 |
| 9 | パフォーマンス | 応答時間3秒以内（95パーセンタイル） | 必須 |
| 10 | 通知・ダッシュボード基盤 | 基本動作確認済 | 推奨 |
| 11 | ドキュメント | 設計書・API仕様書の更新完了 | 推奨 |

**Go/No-Go判定：**

| 判定 | 条件 |
|------|------|
| **Go** | 必須項目（#1-#9）が全て合格 |
| **Conditional Go** | 必須項目に軽微な課題があるが、Phase2開始に影響しないレベル |
| **No-Go** | 必須項目に重大な未解決事項がある場合 |

**判定者：** SC（CIO、管理本部長、情報システム部長）

## 4. ゲートレビュープロセス

### 4.1 レビュー手順

```
Step 1: PM がゲートレビュー資料を準備（レビュー2週間前）
    ↓
Step 2: PMO がチェックリストに基づく事前評価を実施（レビュー1週間前）
    ↓
Step 3: ゲートレビュー会議の開催（全SC委員参加）
    ↓
Step 4: Go/No-Go判定の実施
    ↓
Step 5: 判定結果の記録・通知
    ↓
Step 6: No-Go の場合：是正計画策定 → 再レビュー日程設定
```

### 4.2 ゲートレビュー資料

| # | 資料名 | 内容 | 作成者 |
|---|--------|------|--------|
| 1 | フェーズ完了報告書 | 成果物一覧、進捗サマリー、残課題 | PM |
| 2 | 品質報告書 | テスト結果、メトリクス、障害状況 | QAリーダー |
| 3 | セキュリティ報告書 | 脆弱性診断結果、対応状況 | セキュリティ担当 |
| 4 | UAT結果報告書 | UAT実施結果、業務チーム評価 | 業務チーム |
| 5 | リスク報告書 | 残存リスク、次フェーズリスク | PMO |
| 6 | 次フェーズ計画書 | 次フェーズの詳細計画、リソース計画 | PM |
| 7 | EVM報告書 | コスト・スケジュールの実績vs計画 | PMO |

### 4.3 判定結果の記録

| フィールド | 内容 |
|-----------|------|
| マイルストーンID | M1-M9 |
| レビュー日 | 実施日 |
| 判定結果 | Go / Conditional Go / No-Go |
| 条件付き事項 | Conditional Goの場合の条件 |
| 是正事項 | No-Goの場合の是正計画 |
| 再レビュー日 | No-Goの場合の再レビュー予定日 |
| 判定者署名 | SC全委員の署名 |
| 議事録リンク | Confluence議事録へのリンク |
