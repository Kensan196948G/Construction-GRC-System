## 2. Phase 1: 基盤構築（2026年3月 - 2026年5月）

### 2.1 フェーズ目標

システム全体の技術基盤を構築し、セキュアで拡張性のあるプラットフォームを確立する。

### 2.2 実装スコープ

#### 2.2.1 認証・認可基盤

| 機能 | 詳細 | 優先度 |
|------|------|--------|
| JWT認証 | アクセストークン + リフレッシュトークン方式 | 必須 |
| RBAC実装 | 6ロール（GRC管理者/リスクオーナー/コンプライアンス担当/内部監査員/経営層/一般担当） | 必須 |
| パスワードポリシー | 複雑性要件、有効期限、履歴管理 | 必須 |
| 多要素認証（MFA） | TOTP方式のMFA | 必須 |
| セッション管理 | タイムアウト、同時ログイン制御 | 必須 |
| 監査ログ | 全認証イベントのログ記録 | 必須 |

#### 2.2.2 データベース設計・構築

| 項目 | 詳細 |
|------|------|
| データモデル設計 | 全モジュール共通のマスターデータ、テナント分離設計 |
| マイグレーション基盤 | Django Migrations による版管理 |
| シーディング | マスターデータ、初期ロール、テストデータ |
| インデックス設計 | 主要クエリに対する最適化 |
| パーティショニング | 監査ログテーブルの日付パーティショニング |

#### 2.2.3 API基盤

| 項目 | 詳細 |
|------|------|
| REST API設計 | OpenAPI 3.0仕様準拠 |
| APIバージョニング | URLパスベース（/api/v1/） |
| レート制限 | ロール別レート制限 |
| バリデーション | リクエスト/レスポンスのスキーマバリデーション |
| エラーハンドリング | 統一エラーレスポンスフォーマット |
| API ドキュメント | Swagger UI / ReDoc 自動生成 |

#### 2.2.4 フロントエンド基盤

| 項目 | 詳細 |
|------|------|
| Vue.js 3 プロジェクト構成 | Composition API、TypeScript |
| 状態管理 | Pinia によるストア設計 |
| ルーティング | Vue Router、権限ベースルートガード |
| UIコンポーネント | 共通コンポーネントライブラリ |
| 国際化対応 | vue-i18n（日本語/英語） |

#### 2.2.5 インフラ構築

| 項目 | 詳細 |
|------|------|
| 開発環境 | Docker Compose によるローカル環境 |
| CI/CD | GitHub Actions パイプライン |
| ステージング環境 | 本番同等構成のテスト環境 |
| 監視基盤 | Prometheus + Grafana 初期設定 |
| ログ基盤 | Loki + Promtail 初期設定 |

### 2.3 成果物

| 成果物 | 説明 |
|--------|------|
| 認証・認可モジュール | JWT + RBAC + MFA 実装済み |
| データベーススキーマ | 全テーブル定義 + マイグレーション |
| API基盤 | 共通ミドルウェア + エラーハンドリング |

## 3. Phase 2: リスク管理（2026年5月 - 2026年7月）

### 3.1 フェーズ目標

ISO27001およびNIST CSF 2.0に準拠したリスク管理機能を実装し、建設業特有の安全衛生リスクを統合管理可能にする。

### 3.2 実装スコープ

#### 3.2.1 リスク登録・評価

| 機能 | 詳細 | 優先度 |
|------|------|--------|
| リスク登録 | リスク項目の登録、カテゴリ分類 | 必須 |
| リスク評価 | 影響度 x 発生可能性のマトリクス評価 | 必須 |
| リスクヒートマップ | 視覚的なリスク分布表示 | 必須 |
| リスクオーナー割当 | リスクに対する責任者の割当 | 必須 |
| リスク対応計画 | 回避/軽減/移転/受容の対応計画 | 必須 |
| リスクモニタリング | 定期的なリスク再評価ワークフロー | 必須 |

#### 3.2.2 ISO27001リスク管理

| 機能 | 詳細 |
|------|------|
| 資産台帳管理 | 情報資産の登録・分類・評価 |
| 脅威・脆弱性管理 | 脅威シナリオの管理 |
| 管理策マッピング | 93管理策とリスクの対応付け |
| 適用宣言書（SoA） | Statement of Applicability の生成 |
| リスク対応計画 | 管理策に基づく対応計画 |

#### 3.2.3 安全衛生リスク管理

| 機能 | 詳細 |
|------|------|
| 安全衛生リスクアセスメント | 労安法第28条の2対応 |
| KY（危険予知）活動記録 | 日次KY活動の記録管理 |
| 安全衛生計画管理 | 計画作成・承認・届出管理 |
| 現場別リスク管理 | 建設現場ごとのリスク台帳 |

#### 3.2.4 リスクダッシュボード

| パネル | 内容 |
|--------|------|
| リスクヒートマップ | 全リスクの影響度x発生可能性分布 |
| カテゴリ別集計 | 情報セキュリティ/安全衛生/事業リスク別集計 |
| トレンドグラフ | リスクレベルの推移 |
| 期限超過アラート | 対応期限を超過したリスク一覧 |
| トップ10リスク | 最もスコアの高いリスク一覧 |

### 3.3 成果物

| 成果物 | 説明 |
|--------|------|
| リスク管理モジュール | リスクCRUD + 評価 + 対応計画 |
| ISO27001リスク管理 | 資産台帳 + SoA生成 |
| 安全衛生リスク管理 | リスクアセスメント + KY活動 |
| リスクダッシュボード | 統合リスクビュー |
| リスクレポート | PDF/Excel形式のリスクレポート |

### 3.4 完了条件

- [ ] リスクのCRUD操作が全ロールで正常動作
- [ ] リスクヒートマップが正しく表示
- [ ] ISO27001 93管理策のマッピングが完了
- [ ] 安全衛生リスクアセスメントが動作
- [ ] リスクレポートのPDF出力が正常
- [ ] ユニットテストカバレッジ 80%以上

### 3.5 リソース計画

|------|------|------------|

## 4. Phase 3: コンプライアンス管理（2026年7月 - 2026年9月）

### 4.1 フェーズ目標

建設業法、品確法、労安法を中心とした法令遵守管理機能を実装し、安全パトロールや資格管理を含む包括的なコンプライアンス基盤を構築する。

### 4.2 実装スコープ

#### 4.2.1 法令遵守管理

| 機能 | 詳細 | 優先度 |
|------|------|--------|
| 法令要件台帳 | 建設業法/品確法/労安法の要件一覧管理 | 必須 |
| 遵守状況管理 | 要件ごとの遵守/非遵守/一部遵守のステータス管理 | 必須 |
| 是正管理 | 非遵守項目の是正計画・追跡 | 必須 |
| 法令改正追跡 | 法令改正情報の管理・影響評価 | 必須 |
| コンプライアンスレポート | 法令別・現場別の遵守率レポート | 必須 |

#### 4.2.2 安全パトロール管理

| 機能 | 詳細 |
|------|------|
| パトロール計画 | 年間/月間パトロール計画の策定 |
| チェックリスト管理 | 工種別チェックリストテンプレート |
| パトロール記録 | モバイル対応の記録入力 |
| 写真管理 | 指摘事項の写真記録・紐付け |
| 是正指示・追跡 | 是正指示の発行・期限管理・完了確認 |
| パトロール統計 | 指摘件数推移、カテゴリ分析 |

#### 4.2.3 資格・免許管理

| 機能 | 詳細 |
|------|------|
| 資格台帳 | 作業者別の資格一覧 |
| 有効期限管理 | 更新期限のアラート通知 |
| 配置チェック | 現場作業に必要な資格の配置確認 |
| 資格証画像管理 | 資格証のスキャン画像管理 |
| 教育訓練記録 | 受講履歴・修了証管理 |

#### 4.2.4 協議組織管理

| 機能 | 詳細 |
|------|------|
| 安全衛生協議会管理 | 開催計画・議事録・出席管理 |
| 是正決議管理 | 協議会決議事項の追跡 |
| 施工体制管理 | 施工体制台帳との連携 |

### 4.3 成果物

| 成果物 | 説明 |
|--------|------|
| コンプライアンス管理モジュール | 法令要件管理 + 遵守状況 |
| 安全パトロールモジュール | 計画 + 記録 + 是正管理 |
| 資格管理モジュール | 資格台帳 + 期限管理 |
| モバイル対応画面 | パトロール記録用レスポンシブUI |
| 法令クロスリファレンス | 全法令の関連性マッピング |

### 4.4 完了条件

- [ ] 安全パトロールのモバイル入力が動作
