## 2. Phase1：基盤構築（2026年4月〜9月）

### 2.1 フェーズ目標

システム全体の技術基盤を構築し、セキュアで拡張性のあるプラットフォームを確立する。以降のフェーズが効率的に開発を進められる共通コンポーネントとインフラ環境を整備する。

### 2.2 実装内容詳細

#### 2.2.1 認証・認可基盤

| 機能 | 実装内容 | 技術要素 |
|------|---------|----------|
| ユーザー認証 | ID/パスワード認証、パスワードポリシー管理 | Auth0, bcrypt |
| SSO連携 | SAML 2.0 / OIDC対応、Active Directory連携 | Auth0 Enterprise |
| 多要素認証（MFA） | TOTP、SMS認証、メール認証 | Auth0 MFA |
| RBAC | ロール定義、権限マトリクス、リソースベースアクセス制御 | CASL.js, カスタム実装 |
| セッション管理 | JWTトークン管理、リフレッシュトークン、セッションタイムアウト | JWT, Redis |
| 監査ログ | 全認証イベントのログ記録、不正アクセス検知 | Winston, CloudWatch |

#### 2.2.2 共通マスタ管理

| マスタ | 主要属性 | 機能 |

## 3. Phase2：リスク管理（2026年7月〜12月）

### 3.1 フェーズ目標

ISO31000に準拠したリスクマネジメントプロセスをシステム化し、組織全体のリスクを統合的に管理・可視化する機能を構築する。建設業特有のリスク（工事リスク、安全リスク、環境リスク）にも対応する。

### 3.2 実装内容詳細

#### 3.2.1 リスク台帳管理

| 機能 | 実装内容 | 備考 |
|------|---------|------|
| リスク登録 | リスク名、説明、カテゴリ、発見日、報告者 | 必須項目バリデーション |
| リスク分類 | 多段階カテゴリ分類、タグ付け、カスタム属性 | 建設業カテゴリテンプレート |
| リスク一覧 | フィルタ、ソート、検索、ページネーション | 高速な全文検索 |
| リスク詳細 | 関連情報の集約表示、履歴、コメント | タイムライン表示 |
| リスクインポート | CSV/Excelからの一括取り込み | データバリデーション付き |
| リスクエクスポート | CSV/Excel/PDF出力 | テンプレート選択可 |

#### 3.2.2 リスクアセスメント

| 機能 | 実装内容 | 備考 |
|------|---------|------|
| 定性評価 | 影響度×発生確率のマトリクス評価 | 5段階評価（カスタマイズ可） |
| 定量評価 | 期待損失額、VaR計算 | 金額ベースの評価 |
| ヒートマップ | リスクマップの可視化、ドリルダウン | インタラクティブ |
| リスクスコアリング | 加重スコアリング、スコア自動計算 | 評価基準のカスタマイズ |
| シナリオ分析 | ベスト/ワースト/モストライクリーシナリオ | 比較表示 |
| リスク推移 | スコアの時系列推移、トレンド分析 | 折れ線グラフ |

#### 3.2.3 リスク対応計画

| 機能 | 実装内容 | 備考 |
|------|---------|------|
| 対応策登録 | 対応戦略（回避/軽減/移転/受容）、対応内容 | リスクに紐付け |
| 担当者アサイン | 対応責任者、実施者の設定 | メール通知連携 |
| 進捗管理 | 対応策の進捗率、ステータス管理 | ガントチャート表示 |
| 有効性評価 | 対応策実施後の再評価、残存リスク算定 | ビフォー/アフター比較 |
| コスト管理 | 対応策のコスト見積、実績管理 | 予算対比 |

#### 3.2.4 KRI・モニタリング

| 機能 | 実装内容 | 備考 |
|------|---------|------|
| KRI定義 | 重要リスク指標の定義、閾値設定 | カスタムKRI対応 |
| 自動モニタリング | KRI値の定期取得、閾値監視 | バッチ/リアルタイム |
| アラート | 閾値超過時の自動アラート通知 | メール、Slack、画面通知 |
| ダッシュボード | KRI一覧、トレンド、アラート状況 | リアルタイム更新 |

#### 3.2.5 リスクレポーティング

| 機能 | 実装内容 | 備考 |
|------|---------|------|
| 経営ダッシュボード | トップリスク、リスクプロファイル、KPI | カスタマイズ可能 |
| 定期レポート | 月次・四半期リスクレポートの自動生成 | PDF/Excel出力 |
| アドホックレポート | 条件指定によるカスタムレポート | ドラッグ&ドロップ |
| 建設業特有レポート | 工事別リスク、安全リスク集計 | テンプレート提供 |

### 3.3 成果物一覧

|---|--------|------|

### 3.4 完了条件

- [ ] リスク登録・評価・対応計画の一連のフローが正常動作すること
- [ ] ヒートマップ、KRIダッシュボードが正しく表示されること
- [ ] リスクレポートがPDF/Excelで正常に出力されること
- [ ] リスク管理担当者によるUATが合格すること
- [ ] テストカバレッジ80%以上、セキュリティ脆弱性ゼロ
- [ ] パフォーマンス基準を満たすこと（100件以上のリスク表示が3秒以内）
- [ ] 本番環境へのデプロイが完了し、安定稼働していること

## 4. Phase3：コンプライアンス管理（2026年10月〜2027年3月）

### 4.1 フェーズ目標

ISO27001:2022全93管理策、NIST CSF 2.0、建設業法・品確法・労安法の遵守状況を一元管理し、法改正への迅速な対応を可能にするコンプライアンス管理基盤を構築する。

### 4.2 実装内容詳細

#### 4.2.1 法令管理台帳

| 機能 | 実装内容 | 備考 |
|------|---------|------|
| 法令登録 | 法令名、条文、施行日、所管省庁 | 階層管理（法律→政令→省令） |
| 適用判定 | 組織・事業への適用可否判定 | チェックリスト方式 |
| 遵守状況評価 | 条文ごとの遵守状況セルフチェック | 4段階評価 |
| 法改正追跡 | 法改正情報の登録・影響分析 | RSS/API連携（将来） |
| ギャップ分析 | 遵守状況のギャップ一覧、改善計画 | 優先度付けの自動化 |

#### 4.2.2 ISO27001:2022管理策管理

| 機能 | 実装内容 | 備考 |
|------|---------|------|
| 管理策マスタ | 全93管理策の条文・説明・カテゴリ | 4テーマ（組織・人・物理・技術） |
| SoA管理 | 適用宣言書の作成・管理 | 適用/非適用の判定根拠 |
| 実装状況追跡 | 管理策ごとの実装レベル評価 | 5段階（未実施〜最適化） |
| 証跡管理 | 管理策に対する証跡のアップロード・紐付け | ファイル管理 |
| 有効性評価 | 定期的な有効性測定、指標管理 | 自動リマインダー |
| ギャップレポート | 未実装・不十分な管理策の一覧 | 優先度・リスクスコア連動 |

**ISO27001:2022 93管理策のカテゴリ内訳：**

| テーマ | 管理策数 | 例 |
|--------|---------|-----|
| 組織的管理策 | 37 | 情報セキュリティ方針、脅威インテリジェンス |
| 人的管理策 | 8 | 審査、情報セキュリティ意識向上 |
| 物理的管理策 | 14 | 物理的セキュリティ境界、装置の保守 |
| 技術的管理策 | 34 | アクセス制御、暗号化、ログ管理 |

#### 4.2.3 NIST CSF 2.0対応

| 機能 | 実装内容 | 備考 |
|------|---------|------|
| CSF機能マスタ | 6機能（GV/ID/PR/DE/RS/RC）のカテゴリ・サブカテゴリ | NIST CSF 2.0準拠 |
| 成熟度評価 | 各サブカテゴリの成熟度レベル評価 | ティア1-4 |
| プロファイル管理 | 現状プロファイル、目標プロファイルの管理 | ギャップ可視化 |
| 改善計画 | ギャップに基づく改善アクション管理 | 優先度・期限管理 |
| CSFダッシュボード | レーダーチャート、成熟度推移 | 6機能の可視化 |

**NIST CSF 2.0 6機能：**

| 機能 | 説明 |
|------|------|
| 統治（Govern/GV） | サイバーセキュリティリスク管理戦略の確立・監視 |
| 識別（Identify/ID） | 組織のサイバーセキュリティリスクの理解 |
| 防御（Protect/PR） | サイバーセキュリティリスクを管理するための保護措置 |
| 検知（Detect/DE） | サイバーセキュリティ攻撃・侵害の発見・分析 |
| 対応（Respond/RS） | サイバーセキュリティインシデントへの対応 |
| 復旧（Recover/RC） | サイバーセキュリティインシデントからの復旧 |

## 5. Phase4：監査・レポート（2027年1月〜6月）

### 5.1 フェーズ目標

内部監査プロセスのデジタル化と効率化を実現し、経営レポーティングの自動化により、GRC情報の適時・適切な報告体制を構築する。

### 5.2 実装内容詳細

#### 5.2.1 監査計画管理

| 機能 | 実装内容 |
|------|---------|
| 年間監査計画 | 監査対象の選定、スケジュール策定、リソース配分 |
| リスクベース選定 | リスクスコアに基づく監査対象の優先度判定 |
| 監査スコープ定義 | 監査範囲、監査基準、チェックポイントの定義 |
| 監査チーム編成 | 監査員のアサイン、利益相反チェック |

#### 5.2.2 監査実施支援

| 機能 | 実装内容 |
|------|---------|
| チェックリスト自動生成 | 監査基準に基づくチェックリストの自動生成 |
| 証跡収集 | コンプライアンスモジュールからの自動証跡収集 |
| 監査調書作成 | テンプレートベースの監査調書作成支援 |
| オフライン対応 | 現場での監査用にオフラインモード対応 |
| 写真・添付管理 | 監査時の写真撮影・添付ファイル管理 |

#### 5.2.3 指摘・是正管理

| 機能 | 実装内容 |
|------|---------|

| エスカレーション | 期限超過時の自動エスカレーション |

#### 5.2.4 レポーティング

| 機能 | 実装内容 |

## 6. Phase5：外部連携（2027年4月〜9月）

### 6.1 フェーズ目標

既存の社内システムおよび外部データソースとのシームレスな連携を実現し、データの二重管理を解消する。また、外部向けAPIを提供し、将来のシステム拡張に備える。

#### 6.2.1 ERP連携

| 連携対象 | データ | 方向 | 頻度 | 方式 |

| プロジェクトマスタ | 工事番号、名称、期間 | ERP→GRC | 日次バッチ | REST API |
| 取引先マスタ | 取引先コード、名称 | ERP→GRC | 日次バッチ | REST API |

| 人事システム | 資格情報、研修受講履歴 | 人事→GRC | REST API |
| 勤怠システム | 労働時間データ | 勤怠→GRC | CSV連携 |
