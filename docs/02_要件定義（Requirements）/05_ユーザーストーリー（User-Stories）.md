## 1. ストーリー記述フォーマット

各ユーザーストーリーは以下の形式で記述する。

```
[ロール]として、
[目的/機能]したい、
[理由/価値]のために。
```

**ストーリーポイント基準:**

| SP | 作業規模 | 目安 |
|----|---------|------|
| 1 | 極小 | 設定変更・表示修正レベル |
| 2 | 小 | 単純なCRUD1画面 |
| 3 | 中小 | 条件分岐のあるCRUD |
| 5 | 中 | 複数テーブル連携のCRUD |
| 8 | 大 | ワークフロー・複雑なロジック |
| 13 | 特大 | 複数モジュール横断の大機能 |

## 3. GRC管理者のユーザーストーリー

### EPIC-01: リスク管理

#### US-001: リスクの新規登録

| 項目 | 内容 |
|------|------|
| **ストーリーID** | US-001 |
| **エピック** | EPIC-01 リスク管理 |
| **関連要件** | RSK-001 |
| **ストーリーポイント** | 5 |
| **優先度** | 必須 |
| **フェーズ** | Phase 1 |

**GRC管理者**として、
全社的なリスクを統一フォーマットでリスクレジスターに登録**したい**、
リスク情報の一元管理と全社横断的なリスク可視化を実現する**ために**。

**受入基準:**
- リスクカテゴリ（IT/物理/法務/建設/環境/財務/人的）を選択できる
- リスクIDがカテゴリプレフィックス付きで自動採番される
- 発生可能性（1-5）と影響度（1-5）を入力するとリスクスコアが自動計算される
- リスクレベル（LOW/MEDIUM/HIGH/CRITICAL）が自動判定される
- リスクオーナーをユーザー一覧から選択できる
- 登録後にリスクオーナーに通知が送信される

| **関連要件** | RSK-001 |

**GRC管理者**として、
リスクをカテゴリ・レベル・ステータス・オーナー等で検索・フィルタリング**したい**、
大量のリスク情報から必要な情報を素早く見つける**ために**。

**受入基準:**
- カテゴリ、リスクレベル、ステータス、リスクオーナーでフィルタリングできる
- キーワードによるリスク名称・説明のテキスト検索ができる
- 複数条件の組み合わせ検索ができる
- 検索結果をCSV/Excel形式で出力できる
- フィルタ条件を保存して再利用できる

#### US-003: リスクアセスメントの計画・実施

| 項目 | 内容 |
|------|------|
| **ストーリーID** | US-003 |
| **エピック** | EPIC-01 リスク管理 |
| **関連要件** | RSK-002 |
| **ストーリーポイント** | 8 |
| **優先度** | 必須 |
| **フェーズ** | Phase 1 |

**GRC管理者**として、
リスクアセスメントの計画・実施・記録を一つのワークフローで管理**したい**、
定期的なリスク評価を効率的に実施し、評価漏れを防ぐ**ために**。

**受入基準:**
- アセスメント計画（対象範囲、参加者、スケジュール）を作成できる
- ワークフロー（計画中→実施中→レビュー中→完了→承認済）で管理できる
- 四半期ごとの定期アセスメントを自動スケジューリングできる
- アセスメント結果に基づいてリスクレジスターを一括更新できる
- 参加者への通知・リマインダーが自動送信される
- 承認ワークフロー（リスクオーナー→GRC管理者→経営層）で承認できる

- セルクリックでリスク詳細一覧がポップアップ表示される
- ヒートマップ画像をPNG/PDFで出力できる

**受入基準:**

- 承認/差戻しのワークフローが動作する
- 承認履歴が記録される
