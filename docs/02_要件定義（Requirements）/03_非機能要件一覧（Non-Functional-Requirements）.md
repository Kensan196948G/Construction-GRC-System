## 4. セキュリティ要件（NFR-SEC）

### NFR-SEC-001: 認証方式

| 項目 | 内容 |
|------|------|
| **要件ID** | NFR-SEC-001 |
| **要件名** | 認証方式 |
| **優先度** | 必須 |
| **準拠規格** | ISO27001 A.5.17, A.8.5 |

| 項目 | 要件 |
|------|------|
| 認証方式 | ユーザー名 + パスワード + MFA（TOTP） |
| MFA適用範囲 | 管理者：必須、一般ユーザー：推奨 |
| JWT方式 | アクセストークン（30分）+ リフレッシュトークン（7日） |
| トークン署名 | RS256（RSA 2048bit） |
| アカウントロック | 5回連続失敗で30分ロック |
| パスワード試行制限 | 1分間に最大10回 |
| SSO対応（将来） | SAML 2.0 / OIDC対応設計 |

| ユニーク制約 | ビジネスキー（risk_id, control_id等）にユニーク制約 |
